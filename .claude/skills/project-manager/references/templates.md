# Шаблоны Project Manager

## Шаблон Sprint Report

```markdown
# Sprint Report: [Название спринта]

**Период:** [дата начала] — [дата окончания]
**Ветка:** sprint-X/[название]

## Итоги

| Метрика | Значение |
|---------|----------|
| Задач запланировано | X |
| Задач завершено | Y |
| Completion rate | Z% |
| Velocity | N задач/спринт |

## Завершённые задачи

| ID | Название | Приоритет | Исполнитель |
|----|----------|-----------|-------------|
| issue-1 | Задача 1 | P1 | Developer |

## Незавершённые задачи

| ID | Название | Статус | Причина |
|----|----------|--------|---------|
| issue-2 | Задача 2 | in_progress | Блокер X |

## Блокеры

| Блокер | Влияние | Статус |
|--------|---------|--------|
| Описание | Какие задачи блокирует | Решён/Открыт |

## Tech Debt (новый)

| ID | Описание | Приоритет |
|----|----------|-----------|
| issue-3 | Описание долга | P2 |

## Следующий спринт

- Фокус: [главная цель]
- Перенесённые задачи: [список]
```

## Шаблон Roadmap (базовый)

```markdown
# Roadmap: [Название проекта]

**Версия:** X.Y
**Обновлено:** [дата]

## Q[N] [Год]

### Milestone 1: [Название]
**Цель:** Одно предложение
**Дедлайн:** [дата]
**Статус:** 🔴 Не начат | 🟡 В работе (X%) | 🟢 Завершён

| Epic | Статус | Прогресс |
|------|--------|----------|
| Epic A | ✅ Завершён | 100% |
| Epic B | 🔄 В работе | 60% |
| Epic C | ⏳ Не начат | 0% |

### Milestone 2: [Название]
...

## Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|

## История изменений

| Дата | Изменение |
|------|-----------|
```

## Шаблон Handoff (между сессиями)

```markdown
# Handoff: [Дата]

## Состояние репозитория
- **Ветка:** [название]
- **Последний коммит:** [hash] — [описание]
- **Статус:** Синхронизировано с remote / Есть uncommitted changes

## Активные задачи

### В работе
| ID | Название | Статус | Контекст |
|----|----------|--------|----------|
| issue-1 | Задача | 70% | Где остановились, что дальше |

### Блокеры
| ID | Блокер | Действие |
|----|--------|----------|

## Следующие шаги
1. Первый приоритет
2. Второй приоритет

## Заметки
- Важное наблюдение
- Риск или предупреждение
```

## Чеклист: Открытие сессии

```markdown
## Открытие сессии

### 1. Синхронизация
- [ ] `git pull --rebase`
- [ ] `bd sync --status` — проверить синхронизацию Beads
- [ ] `bd doctor` — проверить health

### 2. Контекст
- [ ] Прочитать `.memory_bank/activeContext.md`
- [ ] Прочитать последний handoff (если есть)
- [ ] `bd list --status=in_progress` — незавершённые задачи

### 3. Планирование
- [ ] `bd ready` — доступные задачи
- [ ] Выбрать задачи на сессию
- [ ] Обновить статусы: `bd update <id> --status=in_progress`

### 4. Готовность
- [ ] `npm run build` — проект собирается
- [ ] `npm run test` — тесты проходят
```

## Чеклист: Открытие спринта

```markdown
## Открытие спринта

### 1. Подготовка
- [ ] Закрыть предыдущий спринт (если не закрыт)
- [ ] `git checkout main`
- [ ] `git pull --rebase`

### 2. Ветка
- [ ] `git checkout -b sprint-[N]/[название]`
- [ ] `git push -u origin sprint-[N]/[название]`

### 3. Планирование
- [ ] Определить цель спринта
- [ ] Выбрать задачи из бэклога
- [ ] Проверить зависимости: `bd blocked`
- [ ] Создать план спринта (см. шаблон Architect)

### 4. Beads
- [ ] Обновить приоритеты задач
- [ ] Создать недостающие задачи
- [ ] `bd sync`

### 5. Memory Bank
- [ ] Обновить `.memory_bank/activeContext.md`
- [ ] Обновить `.memory_bank/progress.md`
```

## Чеклист: Закрытие спринта

```markdown
## Закрытие спринта

### 1. Проверка готовности
- [ ] Все задачи спринта: завершены или перенесены
- [ ] `npm run build` — проект собирается
- [ ] `npm run test` — тесты проходят
- [ ] Code review пройден

### 2. Beads
- [ ] Закрыть завершённые задачи: `bd close <id>`
- [ ] Перенести незавершённые (обновить приоритет если нужно)
- [ ] Зафиксировать tech debt
- [ ] `bd sync`

### 3. Git
- [ ] `git checkout main`
- [ ] `git pull --rebase`
- [ ] Merge sprint branch (PR или локально)
- [ ] `git push`
- [ ] Удалить sprint branch: `git branch -d sprint-[N]/[название]`
- [ ] `git push origin --delete sprint-[N]/[название]`

### 4. Отчётность
- [ ] Создать Sprint Report
- [ ] Обновить Roadmap (прогресс milestone)
- [ ] Обновить `.memory_bank/progress.md`

### 5. Ретро (опционально)
- [ ] Что сделано хорошо?
- [ ] Что можно улучшить?
- [ ] Action items на следующий спринт
```
